# 社区行为准则 | Code of Conduct

## 我们的承诺 | Our Pledge

为了营造一个开放、友好的学术环境，我们作为贡献者和维护者承诺：无论年龄、体型、残疾、族裔、性别认同与表达、经验水平、教育背景、社会经济地位、国籍、外貌、种族、宗教或性取向如何，参与本项目和社区的每个人都应获得无骚扰的体验。

## 我们的标准 | Our Standards

### 积极行为示例 | Examples of Positive Behavior

- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心
- 基于科学证据进行讨论
- 承认他人的贡献和工作
- 保持专业和学术诚信

### 不可接受的行为 | Examples of Unacceptable Behavior

- 使用性化的语言或图像，以及不受欢迎的性关注或挑逗
- 恶意评论、侮辱/贬损性言论，以及人身或政治攻击
- 公开或私下的骚扰
- 未经明确许可，发布他人的私人信息（如物理地址或电子邮件）
- 学术不端行为（剽窃、数据造假等）
- 故意传播错误信息或未经验证的内容
- 其他在专业环境中可被合理认为不适当的行为

## 学术讨论规范 | Academic Discussion Guidelines

作为一个学术导向的社区，我们特别强调：

1. **基于证据**: 所有主张应有文献或数据支持
2. **尊重异议**: 学术争论应针对观点，而非个人
3. **承认不确定性**: 诚实表达知识的局限性
4. **引用规范**: 正确引用他人工作，避免剽窃
5. **建设性批评**: 提出批评时应附带改进建议
6. **开放心态**: 愿意根据新证据调整观点

## 维护者职责 | Maintainer Responsibilities

项目维护者有责任阐明可接受行为的标准，并对任何不可接受的行为采取适当且公平的纠正措施。

维护者有权利和责任删除、编辑或拒绝与本行为准则不符的评论、提交、代码、wiki 编辑、问题和其他贡献，或暂时或永久禁止任何他们认为有不适当、威胁、冒犯或有害行为的贡献者。

## 适用范围 | Scope

本行为准则适用于所有项目空间，包括但不限于：
- GitHub Issues 和 Pull Requests
- GitHub Discussions
- 项目相关的邮件通信
- 线上或线下的项目活动和会议

当个人代表项目或社区时，本准则也适用于公共空间。

## 报告与执行 | Reporting and Enforcement

如果您遇到或目睹违反行为准则的行为，请通过以下方式报告：

- **邮件**: [conduct@example.com]
- **私密 Issue**: 使用 GitHub 的私密报告功能
- **直接联系维护者**: [维护者联系方式]

所有投诉都将被审查和调查，并将根据情况做出必要且适当的回应。项目团队有义务对事件报告者保密。

### 执行措施 | Enforcement Actions

违反行为准则的后果可能包括：

1. **警告**: 私下警告，说明违规行为
2. **临时禁止**: 暂时限制参与项目活动（1-30天）
3. **永久禁止**: 永久禁止参与项目
4. **内容删除**: 删除违规内容
